# Data Shared With Us

This page explains what data is shared with us when system notifications are enabled.

## Why Instance Registration Is Required

System notifications require instance registration so your instance can receive:
- new 4ga Boards version releases (admin only, can be disabled per admin in user preferences)
- important security messages (admin only, when something really requires admin attention)
- other system notifications and poll used to improve 4ga Boards

We do not spam system notifications.

## What Data Is Shared

When system notifications are enabled, your instance shares only the data needed for this flow:
- `instanceUrl`
- `redirectUrl`
- `ipAddress`
- an anonymized user identifier generated by your 4ga Boards instance from `userid`, used so we know that different users responded

## How We Use This Data

We use this data to send system notifications to you 4ga Boards instance and receive poll respones.

## Can You Disable It?

Yes. You can disable system notifications.

If you disable them, we will stop sending regular system notifications and polls. However, admins will still receive important security update notifications.
